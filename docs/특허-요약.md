# 특허 요약 — 2024-0302KR

**디지털 도큐먼트 보안 서비스 제공 방법 및 이를 위한 서버**
(METHOD FOR PROVIDING DIGITAL DOCUMENT SECURITY SERVICE AND SERVER FOR THE SAME)

- 출원인·발명자: **위관우, 류지성** · 대리인: 특허법인비엘티 · 출원일 **2024-06-24** · 청구항 10 · 도면 5

## 배경 / 문제
- PDF 등 디지털 문서는 **무차별 암호 대입 공격에 취약**하고, **한 번이라도 보안이 풀리면** 무방비로 배포·공유되어 보안 문서 기능을 영구히 상실한다(【0005】).
- 해결: **열람 권한을 통합 서버에서 부여**하는 보안 서비스(【0008】) — 복잡한 DRM 없이 안전(【0024】).

## 핵심 메커니즘 — 토큰 회전 (청구항 1)
문서에는 **고유값(해시, 불변)** + **토큰**이 포함된다(【0059】~【0061】). 한 시점에 **유효 토큰은 1개**.

1. 클라이언트 → 서버: 특정 문서 **다운로드 요청** (S310)
2. 서버 → 클라이언트: **고유값 + 제1 토큰**이 포함된 문서 전송 (S320; 결제 완료 후 — 청구항 3)
3. 클라이언트 **뷰어**가 열람 시도 → **제1 토큰**을 서버로 전송 (S330·S340; **API 기반** — 청구항 4)
4. 서버: **토큰 비교로 제1 토큰 인증** (S350)
5. 성공 시: **제1 토큰을 대체할 제2 토큰 생성** (제1≠제2 — 청구항 2) (S370)
6. 서버 → 클라이언트: **열람 개시 명령 + 제2 토큰** 전송 (S380) → 문서 토큰 자동 업데이트 (S390)
   - 인증 실패 시: 열람 개시 명령을 보내지 않음(또는 불가 명령) → 열람 차단 (S360·S365)

> 토큰 = 인증·권한 부여용 데이터(프로세스 식별·유효시간·권한·서명 포함). 서명 무결성은 **고유값(해시)** 으로 확인(【0062】).

## 공유 시 권한 이전 (도 4, 청구항 5~7)
- A(제1 토큰 T1) → B에게 문서 공유(S410). **B가 열람하면 T1→T2로 회전**(서버는 이제 T2 보유).
- 이후 **A가 다시 열려 하면**(보유 토큰 T1) → 서버 보유 T2 ≠ T1 → **인증 실패, 열람 개시 명령 미전송** → A 열람 불가(S430·S440).
- 즉 **마지막에 연 기기만 권한**을 가진다. (B 열람 시 A 온라인이면 *알림* 또는 *A 열람 중지* 가능 — 【0085】)

## 오프라인 / 클라우드
- **오프라인**: 열람 시점에만 온라인이면 토큰 업데이트되고, 이후 **오프라인에서도 열람 가능**(【0078】).
- **클라우드**(도 5): 공용 계정으로 문서 공유하되 **한 번에 한 단말만** 이용, 이용 시마다 토큰 업데이트. 클라우드 사용은 **고과금 정책**(【0090】).

## 서버 구성 (도 2, 청구항 8~10)
- **보안 서비스 제공 서버(100)**: 통신부(110) · 입출력 인터페이스(120) · 메모리(150) · 프로세서(190).
- **플랫폼(PF)**: 웹 서비스(웹사이트/웹앱) + 앱(PC/모바일). **전자 서점**처럼 동작, 문서=도서, 다운로드 비용 표시(【0040】).
- 전송 암호화: 예시로 **128비트 AES**(【0065】, 구현은 비한정).
